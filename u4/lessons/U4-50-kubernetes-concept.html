Container Orchestration (Kubernetes)

Running one container is easy; running hundreds across many machines is not. Kubernetes lets you declare a desired state — "I want 3 replicas running" — and a controller constantly works to make reality match: restarting crashed pods, rescheduling, and scaling on command.
