Distributed Cache (Redis Cluster)

One cache server can only hold so much and handle so much traffic. A distributed cache spreads the data across many nodes: each key is hashed to a node, so the dataset and the load split evenly. Add nodes to scale; replicate so a node failure doesn't lose its slice.
