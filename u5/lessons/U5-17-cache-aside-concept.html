Cache-Aside Pattern

In cache-aside (lazy loading), the application manages the cache. On a read it checks the cache first; on a miss it loads from the database and populates the cache itself. On a write it updates the database and invalidates the stale cache entry. Simple and resilient — the most common pattern.
